# T6M Source Documentation

## Overview

T6M received the majority of the repair and cleanup work during this rebuild process.

Most of the effort focused on stabilizing:

* matchmaking
* profile systems
* messaging
* Steam bridge code
* auth handling
* xNP/PM integration
* backend placeholder systems

The goal was to make the source compile and run without the original backend libraries while keeping the project structure intact for future restoration.

---

## Current Build Direction

The current T6M setup uses standalone local stub systems for missing networking and backend functionality.

This avoids dependency problems caused by missing libraries and unresolved imports such as:

```txt id="rj0m7o"
libpm.lib
__imp__NP_AuthenticateWithDetails
__imp__NP_SendMessage
__imp__NP_Connect
__imp__NP_WriteUserFile
```

The current setup is intended to:

* keep the project compiling
* prevent crashes
* keep systems callable
* allow future backend restoration later

---

## Major Work Completed

### Removed Missing Backend Dependencies

The source originally attempted to link against:

```txt id="b3jow5"
libpm.lib
```

That library was unavailable and caused linker failures.

The dependency was removed from:

* Visual Studio linker settings
* `.vcxproj` dependency lists
* old pragma comments

---

### Rebuilt xNP Stub Layer

A large amount of work went into rebuilding the temporary xNP system.

Temporary replacements were added for functions such as:

```cpp id="34rm0g"
xNP_Connect
xNP_SendMessage
xNP_WriteUserFile
xNP_GetUserFile
xNP_GetPublisherFile
xNP_GetxNPID
xNP_CreateSessionWithCredentials
xNP_SetRichPresence
xNP_StoreRichPresence
```

These now return safe placeholder values instead of crashing or generating unresolved externals.

---

### Fixed Authentication Conflicts

The source repeatedly conflicted between:

```cpp id="2lcw7n"
NPAuthenticateResult
xNPAuthenticateResult
```

and:

```cpp id="y4n77k"
NPAsync
xNPAsync
```

The standalone build direction now consistently uses:

```cpp id="s1nq2v"
xNPAuthenticateResult
xNPAsync<xNPAuthenticateResult>
```

inside the local stub system.

This removed:

* enum conflicts
* template conflicts
* redefinition errors
* unresolved externals

---

### Simplified Auth Result Handling

Original auth code depended on enums such as:

```cpp id="zb9cl5"
AuthenticateResultOK
AuthenticateResultBadDetails
AuthenticateResultAlreadyLoggedIn
```

These definitions conflicted with multiple headers and rebuild attempts.

The temporary standalone auth system now simply uses:

```cpp id="h7rf4m"
0 = success
```

and checks:

```cpp id="6f0yfc"
if (result->result != 0)
```

This keeps the source compiling while the real backend is unavailable.

---

### Fixed PM_SendMessage Problems

A major issue involved:

```cpp id="t8jdyj"
PM_SendMessage
```

The source originally:

* included `.cpp` files directly
* had duplicate bodies
* had mismatched signatures
* mixed C and C++ linkage

The temporary solution now uses:

* a normal header declaration
* a simple placeholder implementation
* a clean 3-argument function signature

Example:

```cpp id="zh9a5h"
int __cdecl PM_SendMessage(
    PMID onlineID,
    uint8_t* message,
    int messageLength
)
```

The full PM/message bridge is still unfinished.

---

### Fixed bdProfile.cpp

`bdProfile.cpp` accidentally contained duplicated pasted code.

This caused:

* duplicate definitions
* multiple body errors
* compile instability

The duplicate implementation was removed.

---

### Fixed Profile Structures

The project expected:

```cpp id="cm3y9m"
profiles->results[i].xNPID
```

So the profile result structures were rebuilt to match the expected layout.

Example:

```cpp id="f4l8mh"
struct PMGetPT5ProfileDataEntry
{
    xNPID xNPID;
    char* data;
};
```

---

### Fixed Rich Presence Overloads

Steam friends/profile code required both versions of:

```cpp id="hz3pm8"
xNP_GetFriendRichPresence(id)
xNP_GetFriendRichPresence(id, key)
```

Matching overloads were added so the project compiles correctly.

---

### Stabilized Matchmaking Replies

Several matchmaking functions originally depended on unfinished backend systems.

Temporary placeholder replies were added to functions such as:

```cpp id="lm22yq"
inviteto_match_session
submit_performance
get_performance_values
delete_match_session
```

This prevents crashes while backend systems remain unfinished.

## Known Issues

### PM / Messaging Layer

The PM/message bridge still requires future cleanup.

Areas still unfinished include:

```cpp id="dz6c4u"
PM_SendMessage
xNP_SendMessage
NP_SendMessage
dw_messaging_send_global_im
```

The current recommendation is:

* leave placeholder stubs in place
* avoid wiring into real networking yet
* revisit after the project fully stabilizes

## Current Status

T6M currently:

* builds using standalone stubs
* avoids missing libpm/libnp dependencies
* has placeholder auth
* has placeholder messaging
* has placeholder matchmaking
* has placeholder profile handling
* has reduced Steam bridge conflicts
* still has unfinished PM/message systems

---

## Notes for GitHub

This source is currently in a rebuild/cleanup phase.

Many backend systems are temporarily stubbed because the original private libraries and services are unavailable. The current goal is source stabilization and compile recovery before restoring full backend functionality.
